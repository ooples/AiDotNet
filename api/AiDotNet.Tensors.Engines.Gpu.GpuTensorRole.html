Namespace
AiDotNet.Tensors.Engines.Gpu

AiDotNet.Tensors.dll

Defines the role of a GPU-resident tensor for memory management and optimization decisions.

public enum GpuTensorRole

Fields

Activation = 4

Activation tensor produced during forward pass. May be kept for backward pass in training mode.

AttentionCache = 9

Attention cache for transformer models (KV cache). These can be large and benefit from persistent GPU residency.

Bias = 3

Bias tensor that persists across forward passes. These are candidates for permanent GPU residency.

Constant = 10

Constant tensor that doesn't change (e.g., lookup tables, fixed masks).

General = 0

General-purpose tensor with no specific role.

Gradient = 6

Gradient tensor produced during backward pass.

Input = 1

Input tensor that will be uploaded from CPU once and reused.

Intermediate = 5

Intermediate tensor that is temporary within an operation. Can be released immediately after use.

OptimizerState = 11

Optimizer state tensor (momentum, velocity, accumulated gradients, etc.). These are persistent across training steps and should remain GPU-resident.

Output = 7

Output tensor that will eventually be downloaded to CPU.

Statistics = 8

Normalization statistics (mean, variance, etc.). May be kept for inference or updated during training.

Weight = 2

Weight tensor that persists across forward passes. These are candidates for permanent GPU residency.
