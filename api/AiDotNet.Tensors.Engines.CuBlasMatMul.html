Remarks

Performance Target: ~30,000 GFLOPS

cuBLAS uses column-major storage (Fortran-style), while C#/.NET uses row-major. For row-major C = A × B, we compute via cuBLAS: C^T = B^T × A^T Since row-major C is stored the same as column-major C^T, we get correct results.

Usage for DenseLayer: 

// DenseLayer forward: output = input @ weights.T + bias
// input: [batch, in_features]
// weights: [out_features, in_features]
// weights.T: [in_features, out_features]
// output: [batch, out_features]

using var cuBlas = new CuBlasMatMul();
var output = cuBlas.MatMul(input, weightsTransposed);

Remarks

This is the key optimization for DenseLayer:

- Weights (typically large, e.g., 768×3072 = 9MB) stay on GPU - Only activations (much smaller per batch) transfer each forward pass - Eliminates 99%+ of memory transfer overhead
