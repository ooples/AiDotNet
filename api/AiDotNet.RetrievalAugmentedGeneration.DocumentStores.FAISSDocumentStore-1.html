Core logic for adding multiple vector documents in a batch with automatic indexing.

protected override void AddBatchCore(IList<VectorDocument<T>> vectorDocuments)

Parameters

vectorDocuments IList<VectorDocument<T>>

The validated list of vector documents to add.

Remarks

Batch operations are more efficient than adding documents individually. Each document receives a sequential integer index for fast vector lookup.

For Beginners: Adding many documents at once is faster.

Instead of:

// Slow - one at a time
foreach (var doc in docs)
    store.Add(doc);

Do this:

// Fast - all at once
store.AddBatch(docs);

Exceptions

ArgumentException

Thrown when a document's embedding has inconsistent dimensions.

Clear()

Removes all documents from the store and resets all indices.

public override void Clear()

Remarks

Clears all documents, resets the vector dimension to 0, resets the index counter, and clears the vector index. The store is returned to its initial empty state.

For Beginners: Completely empties the store.

After calling Clear():

  • All documents are gone
  • Vector dimension resets to 0
  • Index counter resets to 0
  • Ready to accept new documents

Use with caution - this cannot be undone!

Example:

store.Clear();
Console.WriteLine($"Documents remaining: {store.DocumentCount}"); // 0

GetAllCore()

Core logic for retrieving all documents in the store.

protected override IEnumerable<Document<T>> GetAllCore()

Returns

IEnumerable<Document<T>>

An enumerable of all documents without their vector embeddings.

Remarks

Returns all documents in the store in no particular order. The vector embeddings are not included in the results - only the document metadata and content.

For Beginners: Gets every document in the store.

Use cases:

  • Export all documents to a file
  • Migrate to a different document store
  • Bulk processing or analysis
  • Debugging - see what's stored

Warning: For large collections (> 10K documents), this can use significant memory. Consider pagination for production systems with large datasets.

Example:

// Get all documents
var allDocs = store.GetAll();
Console.WriteLine($"Total documents: {allDocs.Count()}");

// Export to JSON
var json = JsonConvert.SerializeObject(allDocs);
File.WriteAllText("backup.json", json);

GetSimilarCore(Vector<T>, int, Dictionary<string, object>)

Core logic for similarity search using cosine similarity with optional metadata filtering.

protected override IEnumerable<Document<T>> GetSimilarCore(Vector<T> queryVector, int topK, Dictionary<string, object> metadataFilters)

Parameters

queryVector Vector<T>

The validated query vector.

topK int

The validated number of documents to return.

metadataFilters Dictionary<string, object>

The validated metadata filters.

Returns

IEnumerable<Document<T>>

Top-k similar documents ordered by cosine similarity score.

Remarks

Searches through all documents, filters by metadata, computes cosine similarity scores, and returns the top-k most similar documents ordered by descending similarity.

For Beginners: Finds documents most similar to your query.

Process:

  1. Filter documents by metadata (if filters provided)
  2. Calculate similarity between query and each document
  3. Sort by similarity score (highest first)
  4. Return top-k matches

Example:

// Find 5 most similar documents
var results = store.GetSimilar(queryVector, topK: 5);

// Find 10 similar documents from 2024
var filters = new Dictionary<string, object> { ["year"] = "2024" };
var filtered = store.GetSimilarWithFilters(queryVector, 10, filters);
