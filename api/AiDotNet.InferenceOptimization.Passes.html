Namespace AiDotNet.InferenceOptimization.Passes

Classes

AlgebraicSimplificationPass<T>

Applies algebraic simplification rules to reduce computational complexity. Examples:

  • x * 1 = x
  • x + 0 = x
  • x * 0 = 0
  • x / 1 = x
  • x^1 = x
  • x^0 = 1
CommonSubexpressionEliminationPass<T>

Eliminates common subexpressions by sharing computation. If the same operation with the same inputs appears multiple times, compute it once and reuse the result.

ConstantFoldingPass<T>

Folds constant expressions at compile time to reduce runtime computation. For example: If two constants are multiplied, compute the result once during optimization rather than every inference call.

ConvBatchNormFusionPass<T>

Fuses Convolution + BatchNormalization into a single operation. This is a critical optimization that eliminates the normalization overhead during inference.

ConvBatchNormReLUFusionPass<T>

Fuses Convolution + BatchNormalization + ReLU into a single operation. This is one of the most common patterns in CNNs (ResNet, VGG, etc.) and provides significant speedup by reducing memory traffic and kernel launches.

DeadCodeEliminationPass<T>

Eliminates nodes that don't contribute to the output (dead code). This includes nodes with no consumers and nodes that are not reachable from outputs.

ElementwiseFusionPass<T>

Fuses consecutive elementwise operations into a single operation. For example: (x + y) * z can be computed in a single fused kernel. This reduces memory bandwidth by avoiding intermediate results.

InPlaceOptimizationPass<T>

Marks operations that can be performed in-place to reduce memory allocation. Operations like ReLU, Dropout, and some elementwise operations can modify their input tensors directly instead of allocating new memory.

LayoutOptimizationPass<T>

Optimizes tensor layout (NCHW vs NHWC) for better hardware utilization. Different hardware architectures prefer different layouts:

  • NCHW: Better for GPUs (NVIDIA)
  • NHWC: Better for some CPUs and TPUs
MatMulBiasActivationFusionPass<T>

Fuses MatMul + Bias + Activation (ReLU, GELU, etc.) into a single operation. This is the most common pattern in transformer feed-forward networks and MLPs.

MatMulBiasFusionPass<T>

Fuses MatMul + Bias (Add) operations into a single Gemm operation. This is extremely common in fully connected layers and transformers.

MemoryReuseOptimizationPass<T>

Optimizes memory usage by reusing buffers for different operations. Analyzes the lifetime of tensors and assigns memory pools to reduce overall memory footprint during inference.

MultiHeadAttentionFusionPass<T>

Fuses multi-head attention components into a single optimized operation. Multi-head attention consists of multiple Q, K, V projections, attention computation, and output projection. Fusing these provides significant speedup in transformers.

OptimizationPassBase<T>

Base class for optimization passes.

StrengthReductionPass<T>

Replaces expensive operations with cheaper equivalent operations. Examples:

  • x^2 -> x * x
  • x * 2 -> x + x
  • x / 2 -> x * 0.5
  • sqrt(x^2) -> abs(x)

Interfaces

IOptimizationPass<T>

Interface for optimization passes that transform computation graphs to improve inference performance.
