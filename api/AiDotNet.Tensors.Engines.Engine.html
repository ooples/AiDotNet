Provides automatic hardware-accelerated engine selection. Similar to how SIMD automatically selects Vector128/256/512 based on CPU features, this factory automatically selects GPU or CPU engine based on available hardware.

Remarks

Priority Order (highest to lowest):

  1. DirectGpu - Custom kernels with CLBlast fallback (best performance)
  2. CPU - Always available with SIMD acceleration

The Default property lazily initializes on first access and probes for GPU availability. This ensures optimal performance without manual configuration.

Default

Gets the default engine instance, automatically selected based on available hardware. First access detects GPU availability and creates the optimal engine. Subsequent accesses return the cached instance. Thread-safe via double-checked locking pattern.

public static IEngine Default { get; }

Property Value

IEngine

GpuEngine if GPU is available and initialized successfully, otherwise CpuEngine.

Examples

// Automatic engine selection - no configuration needed
var vector = new Vector<float>(1000);
vector.ApplyInPlace(x => x * 2, Engine.Default);

// Check what was selected
Console.WriteLine(Engine.Default.Name);
// Output: "GPU Engine (NVIDIA RTX 4090)" or "CPU Engine"

DirectGpu

Gets the DirectGpu engine for high-performance GPU operations. This provides direct access to optimized custom kernels for matrix operations. Works on ALL .NET versions including .NET Framework 4.6.2.

public static DirectGpuEngine? DirectGpu { get; }

Property Value

DirectGpuEngine

Examples

// Direct GPU matrix multiplication
var engine = Engine.DirectGpu;
if (engine != null && engine.IsAvailable)
{
    float[]? result = engine.MatMul(A, B, M, K, N);
}

Remarks

The DirectGpuEngine provides:

  • Optimized GEMM with hierarchical tiling and double-buffering
  • Fused operations (GEMM+Bias+Activation) to eliminate memory round-trips
  • Float32-only kernels for maximum GPU performance
  • Generic type support via boundary conversion
  • Pure P/Invoke - no managed GPU runtime dependency
