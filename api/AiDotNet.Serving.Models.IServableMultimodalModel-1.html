Interface for multimodal models that can be served via the REST API. Multimodal models handle multiple input types (text, images, audio) and produce embeddings in a shared vector space.

Remarks

This interface extends the standard IServableModel<T> with capabilities for handling multiple modalities, particularly useful for models like CLIP that can encode both text and images into comparable embeddings.

For Beginners: Multimodal models understand multiple types of data:

  • Text: "A photo of a cat" -> [0.2, 0.5, 0.1, ...]
  • Image: (cat.jpg) -> [0.21, 0.48, 0.12, ...]

Because both outputs are in the same "space", you can compare them directly:

  • High similarity = text describes the image well
  • Low similarity = text doesn't match the image

This enables powerful applications like image search using text queries.

Both text and image embeddings will have this same dimension. Common values are 512 (CLIP ViT-B/32) or 768 (CLIP ViT-L/14).
