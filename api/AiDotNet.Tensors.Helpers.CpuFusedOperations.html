Remarks

Unlike GPU fused operations which primarily eliminate kernel launch overhead, CPU fused operations provide performance benefits through:

  • Cache locality: Processing data in a single pass keeps it in L1/L2 cache
  • Reduced allocations: No intermediate tensors needed between operations
  • SIMD vectorization: Uses TensorPrimitives for hardware-accelerated operations
  • Tiling: Cache-efficient blocking for large matrix operations

Performance Characteristics:

  • FusedGemmBiasActivation: 20-40% faster than separate operations
  • FusedLayerNorm: 30-50% faster due to single-pass mean/variance calculation
  • FusedResidualLayerNorm: 40-60% faster for Transformer blocks
