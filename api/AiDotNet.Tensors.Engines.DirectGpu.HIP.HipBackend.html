Namespace
AiDotNet.Tensors.Engines.DirectGpu.HIP

HIP backend for AMD GPUs with real MFMA (Matrix Core) acceleration. Automatically selects the optimal kernel based on GPU architecture:

  • CDNA (MI100/200/300): Full MFMA with wave64
  • RDNA3 (RX 7000): WMMA with wave32
  • RDNA2 (RX 6000): Optimized scalar with wave32
public sealed class HipBackend : IDirectGpuBackend, IDisposable

Remarks

Performance Targets:

  • MI200: 25,000+ GFLOPS (full MFMA)
  • MI100: 20,000+ GFLOPS (MFMA)
  • RX 7900 XTX: 15,000+ GFLOPS (WMMA)
  • RX 6800 XT: 8,000+ GFLOPS (optimized scalar)

BatchedGemm(IGpuBuffer, IGpuBuffer, IGpuBuffer, int, int, int, int, float, float)

Batched matrix multiplication for many small matrices: C[i] = alpha * A[i] * B[i] + beta * C[i]

public void BatchedGemm(IGpuBuffer A, IGpuBuffer B, IGpuBuffer C, int M, int N, int K, int batchCount, float alpha = 1, float beta = 0)

Parameters

A IGpuBuffer

GPU buffer containing all A matrices concatenated (batchCount * M * K elements).

B IGpuBuffer

GPU buffer containing all B matrices concatenated (batchCount * K * N elements).

C IGpuBuffer

GPU buffer for output C matrices concatenated (batchCount * M * N elements).

M int

Rows of each A matrix and each C matrix.

N int

Columns of each B matrix and each C matrix.

K int

Columns of each A matrix and rows of each B matrix.

batchCount int

Number of matrix multiplication operations to perform.

alpha float

Scalar multiplier for A*B (default 1.0).

beta float

Scalar multiplier for C (default 0.0).

Remarks

Uses work-stealing persistent kernel for efficient processing of many small matrix multiplications.

All matrices within a batch must have the same dimensions (M x K for A, K x N for B).

Matrices are stored concatenated in memory: A[0], A[1], ..., A[batchCount-1].
