Remarks

Purpose:

The registry provides centralized management of GPU memory for tensors. It tracks which tensors are currently on GPU and can make eviction decisions when memory pressure is detected.

Eviction Strategy:

  • Weights/Biases are never evicted (permanent residents)
  • Activations are evicted first (can be recomputed)
  • Intermediates are evicted next (temporary by definition)
  • LRU ordering within each priority tier
