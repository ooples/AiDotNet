Remarks

The scheduler determines which sequences should be processed in each iteration, balancing priorities, fairness, and memory constraints.

For Beginners: The scheduler is like a restaurant host.

When new customers arrive (requests), the host must decide:

  • Who gets seated next? (priority)
  • How many can we serve at once? (batch size)
  • Do we have enough tables/kitchen capacity? (memory/compute)
  • Should we pause someone's meal to serve urgent customers? (preemption)

The scheduler makes these decisions to maximize throughput while ensuring fairness and meeting priority requirements.
