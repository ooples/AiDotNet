Remarks

CURE (Clustering Using REpresentatives) is a hierarchical clustering algorithm that represents each cluster by a set of well-scattered representative points, which are then shrunk toward the cluster center.

For Beginners: CURE is designed to find non-spherical clusters: - Uses multiple "representative" points per cluster instead of just a center - These points are pulled slightly toward the center (shrinking) - This helps find oddly-shaped clusters (like bananas or spirals)

It's like describing a cluster by several key locations within it, rather than just one center point.

Property Value

int

Number of representatives. Default is 5. More representatives = better capture of cluster shape but slower.

Remarks

For Beginners: Representatives are scattered points that describe the cluster's shape. More representatives can capture complex shapes but increase computation time.

SampleFraction

Gets or sets the sample fraction for large datasets.

public double SampleFraction { get; set; }

Property Value

double

Fraction of data to sample. Default is 1.0 (use all data). For very large datasets, try 0.1-0.5.

ShrinkFactor

Gets or sets the shrink factor for representative points.

public double ShrinkFactor { get; set; }

Property Value

double

Shrink factor between 0 and 1. Default is 0.3. 0 = no shrinking, 1 = all representatives at center.

Remarks

For Beginners: The shrink factor controls how much representatives are pulled toward the center: - Low value (0.1): Representatives stay near edges (good for chains) - High value (0.5): Representatives move more toward center - Too high: Loses cluster shape information

UsePartitioning

Gets or sets whether to use random partitioning.

public bool UsePartitioning { get; set; }

Property Value

bool

When true, randomly partition data before clustering. Default is false.

Remarks

Random partitioning can speed up clustering of large datasets by processing partitions independently, then merging.
