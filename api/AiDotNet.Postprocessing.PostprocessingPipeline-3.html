Remarks

A postprocessing pipeline applies transformers in sequence, passing the output of each transformer as input to the next. This enables composable postprocessing workflows similar to sklearn's Pipeline, applied to model outputs.

For Beginners: Think of a pipeline as a series of steps for processing model output. For example: 1. First, apply softmax to get probabilities 2. Then, decode indices to labels 3. Finally, format the output

The pipeline ensures all these steps happen in the right order.
