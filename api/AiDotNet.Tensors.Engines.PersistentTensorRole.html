Specifies the role of a persistent tensor for GPU memory management hints.

public enum PersistentTensorRole

Fields

AttentionCache = 4

Attention key/value caches for inference. Used in autoregressive generation to cache previous attention states.

Biases = 1

Layer biases that change only during training updates.

Constant = 6

Constant tensors that never change (e.g., precomputed frequencies, positional encodings).

Embeddings = 3

Embedding lookup tables. These can be very large and benefit significantly from GPU persistence.

NormalizationParams = 2

Normalization parameters (gamma/beta for BatchNorm, LayerNorm, etc.).

OptimizerState = 5

Optimizer state tensors (velocity, momentum, etc.).

Other = 7

Other persistent tensors not fitting above categories.

Weights = 0

Layer weights that change only during training updates. These are the primary candidates for GPU persistence.

Remarks

Purpose:

When a tensor is registered as persistent, the engine may choose to keep it resident in GPU memory across multiple operations, avoiding repeated CPU-GPU transfers.

Performance Impact:

Without persistence: Transfer weights (e.g., 285MB) every forward pass. With persistence: Transfer weights once, only activations per pass. Expected speedup: 100-1000x for weight-heavy layers.
