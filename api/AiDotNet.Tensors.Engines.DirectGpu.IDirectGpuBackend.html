Interface for direct GPU backend implementations (OpenCL, CUDA, etc.). All operations use float32 for optimal GPU performance.

public interface IDirectGpuBackend : IDisposable

Remarks

Design Philosophy:

This interface abstracts vendor-specific GPU implementations while enforcing float32-only operations for maximum performance. Generic type conversion happens at a higher level (DirectGpuEngine), keeping backends simple and fast.

Performance Targets:

  • GEMM: 10,000+ GFLOPS (10-100x faster than CLBlast)
  • Fused ops: 20-50% faster than separate operations
  • Memory: GPU-resident buffers to eliminate transfer overhead

Remarks

Uses work-stealing persistent kernel for efficient processing of many small matrix multiplications.

All matrices within a batch must have the same dimensions (M x K for A, K x N for B).

Matrices are stored concatenated in memory: A[0], A[1], ..., A[batchCount-1].
