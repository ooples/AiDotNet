Interface IDirectGpuBackend

Namespace
AiDotNet.Tensors.Engines.DirectGpu
Assembly
AiDotNet.Tensors.dll

Interface for direct GPU backend implementations (OpenCL, CUDA, etc.). All operations use float32 for optimal GPU performance.

public interface IDirectGpuBackend : IDisposable
Inherited Members
IDisposable.Dispose()

Remarks

Design Philosophy:

This interface abstracts vendor-specific GPU implementations while enforcing float32-only operations for maximum performance. Generic type conversion happens at a higher level (DirectGpuEngine), keeping backends simple and fast.

Performance Targets:

  • GEMM: 10,000+ GFLOPS (10-100x faster than CLBlast)
  • Fused ops: 20-50% faster than separate operations
  • Memory: GPU-resident buffers to eliminate transfer overhead

BatchedGemm(IGpuBuffer, IGpuBuffer, IGpuBuffer, int, int, int, int, float, float)

Batched matrix multiplication for many small matrices: C[i] = alpha * A[i] * B[i] + beta * C[i]

void BatchedGemm(IGpuBuffer A, IGpuBuffer B, IGpuBuffer C, int M, int N, int K, int batchCount, float alpha = 1, float beta = 0)

Parameters

A IGpuBuffer

GPU buffer containing all A matrices concatenated (batchCount * M * K elements).

B IGpuBuffer

GPU buffer containing all B matrices concatenated (batchCount * K * N elements).

C IGpuBuffer

GPU buffer for output C matrices concatenated (batchCount * M * N elements).

M int

Rows of each A matrix and each C matrix.

N int

Columns of each B matrix and each C matrix.

K int

Columns of each A matrix and rows of each B matrix.

batchCount int

Number of matrix multiplication operations to perform.

alpha float

Scalar multiplier for A*B (default 1.0).

beta float

Scalar multiplier for C (default 0.0).

Remarks

Uses work-stealing persistent kernel for efficient processing of many small matrix multiplications.

All matrices within a batch must have the same dimensions (M x K for A, K x N for B).

Matrices are stored concatenated in memory: A[0], A[1], ..., A[batchCount-1].
