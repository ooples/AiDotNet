Defines an interface for computing distance or similarity between vectors.

public interface IDistanceMetric<T>

Type Parameters

T

The numeric type used for calculations (e.g., float, double).

Remarks

Distance metrics are fundamental to many clustering algorithms as they define what it means for two points to be "similar" or "close together". Different metrics are suitable for different types of data and applications.

For Beginners: A distance metric tells us how to measure "how far apart" two data points are.

Common examples:

  • Euclidean: Straight-line distance (what you'd measure with a ruler)
  • Manhattan: City-block distance (sum of differences along each axis)
  • Cosine: Angle between vectors (useful for text/documents)

The choice of distance metric can significantly affect clustering results.

Compute(Vector<T>, Vector<T>)

Computes the distance between two vectors.

T Compute(Vector<T> a, Vector<T> b)

Parameters

a Vector<T>

The first vector.

b Vector<T>

The second vector.

Returns

T

The computed distance. Always non-negative for valid metrics.

Remarks

For Beginners: This calculates how far apart two points are.

For example, with Euclidean distance:

  • Points [0, 0] and [3, 4] have distance 5 (Pythagorean theorem)
  • Points [1, 1] and [1, 1] have distance 0 (same point)

Different metrics will give different values for the same pair of points.

Exceptions

ArgumentException

Thrown if vectors have different lengths.

ComputePairwise(Matrix<T>)

Computes the full pairwise distance matrix between all rows.

Matrix<T> ComputePairwise(Matrix<T> data)

Parameters

data Matrix<T>

A matrix where each row is a data point.

Returns

Matrix<T>

A symmetric matrix where element [i, j] is the distance between row i and row j. The diagonal is always zero.

Remarks

This operation has O(n^2) complexity where n is the number of rows. For large datasets, consider using spatial indexing structures like KD-Trees.

For Beginners: This creates a table showing the distance between every pair of points. It's like a mileage chart showing distances between cities.

The result is symmetric (distance A to B equals distance B to A) and the diagonal is all zeros (distance from a point to itself is zero).
