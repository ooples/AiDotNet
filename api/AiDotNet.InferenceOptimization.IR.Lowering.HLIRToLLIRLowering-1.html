Remarks

Design Philosophy:

The lowering process transforms high-level semantic operations into low-level hardware-optimized operations. This is similar to MLIR's progressive lowering or TVM's Relay to TIR conversion.

Lowering Stages:

  1. Type conversion: Generic T -> concrete types (float32, etc.)
  2. Operation mapping: Semantic ops -> hardware ops
  3. Layout selection: Choose optimal memory layouts
  4. Scheduling: Generate schedules for loops
  5. Memory planning: Allocate and reuse buffers

Exceeds Standards By:

  • Preserves fusion information from HLIR
  • Automatic algorithm selection (Winograd, FFT, etc.)
  • Multi-device lowering in single pass
  • Quantization-aware lowering
