Remarks

For Beginners: ExperimentTracker stores all your experiment data locally in a structured format. It's like having your own MLflow server without needing to set up any infrastructure.

Directory structure:

tracking_uri/
|-- experiments/
|   |-- experiment_name/
|   |   |-- metadata.json
|   |   |-- runs/
|   |   |   |-- run_id/
|   |   |   |   |-- info.json
|   |   |   |   |-- params.json
|   |   |   |   |-- metrics/
|   |   |   |   |   |-- metric_name.json
|   |   |   |   |-- artifacts/

Example usage:

// Create tracker
var tracker = new ExperimentTracker("./mlruns");

// Set experiment
tracker.SetExperiment("image-classification");

// Start a run
var run = tracker.StartRun("baseline-resnet50");

// Log parameters
tracker.LogParameters(new Dictionary<string, string>
{
    ["learning_rate"] = "0.001",
    ["batch_size"] = "32",
    ["epochs"] = "100"
});

// During training
for (int epoch = 0; epoch < 100; epoch++)
{
    var (loss, accuracy) = TrainEpoch();
    tracker.LogMetrics(new Dictionary<string, double>
    {
        ["loss"] = loss,
        ["accuracy"] = accuracy
    }, step: epoch);
}

// Log model
tracker.LogModel("./model.onnx", "best_model");

// End run
tracker.EndRun();

Security Note: For enhanced security, specify the full path to git rather than relying on PATH resolution. Common locations:

  • Linux/macOS: /usr/bin/git
