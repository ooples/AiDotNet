Remarks

These metrics are essential for evaluating the quality of segmentation models on 3D data like point cloud segmentation, mesh segmentation, and voxel classification.

For Beginners: When a model predicts labels for different parts of a 3D shape (like "leg", "seat", "back" for a chair), these metrics tell us how accurate those predictions are compared to the ground truth labels.

MeanIoU(int[], int[], int, int)

Computes Mean Intersection over Union (mIoU) for segmentation.

public static double MeanIoU(int[] predictions, int[] groundTruth, int numClasses, int ignoreIndex = -1)

Parameters

predictions int[]

The predicted class labels as integers.

groundTruth int[]

The ground truth class labels as integers.

numClasses int

The total number of classes.

ignoreIndex int

Optional class index to ignore (e.g., for unlabeled regions). Default is -1 (none).

Returns

double

The mIoU value in range [0, 1]. Higher is better (1 = perfect).

Remarks

mIoU computes IoU (intersection/union) for each class and averages them. It's the standard metric for semantic segmentation tasks.

For Beginners: IoU measures how much the predicted region overlaps with the true region. mIoU averages this across all classes: - mIoU = 1.0: Perfect segmentation - mIoU > 0.7: Good segmentation - mIoU 0.5-0.7: Acceptable segmentation - mIoU < 0.5: Poor segmentation

Exceptions

ArgumentException

Thrown when arrays have different lengths.
